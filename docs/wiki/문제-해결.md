# 문제 해결

**"안 된다!"라고 생각할 때 확인할 것들입니다. 대부분의 문제는 간단하게 해결됩니다.**

---

## Git Tab이 안 켜져요

### 증상
- Git Tab 아이콘을 클릭했는데 아무것도 안 뜹니다.
- 또는 창이 나타났다가 바로 닫힙니다.

### 해결법

#### 방법 1: .NET 8 런타임 확인

Git Tab은 **.NET 8 Desktop Runtime**이 필요합니다. 설치되어 있는지 확인하세요.

**1. 확인하기:**
1. Windows 검색창(🔍)에서 **"프로그램"** 또는 **"Programs and Features"** 입력.
2. 설치된 프로그램 목록에서 **".NET Desktop Runtime"**을 찾습니다.
3. **"8.0"** 버전이 있으면 설치돼 있는 것입니다.

**2. 설치되지 않았다면:**
1. [Microsoft .NET 다운로드 페이지](https://dotnet.microsoft.com/download/dotnet/8.0)로 이동합니다.
2. **".NET 8.0"** → **"Desktop Runtime"** 선택.
3. **"x64"** 다운로드 (보통 64비트 Windows 사용).
4. 파일을 실행해 설치합니다.
5. 설치 완료 후 Git Tab을 다시 실행합니다.

#### 방법 2: 관리자 권한으로 실행

1. Git Tab 바로가기(또는 실행 파일)를 우클릭합니다.
2. **"관리자 권한으로 실행(Run as administrator)"** 선택.

#### 방법 3: 재설치

1. Windows "제어판" → "프로그램 제거" 또는 "Program and Features" 열기.
2. "Git Tab" 찾아 **"제거(Uninstall)"** 클릭.
3. [시작하기](Getting-Started) 페이지에 따라 다시 설치합니다.

## "Git을 찾을 수 없다"는 오류가 나요

### 증상
```
Git command not found
Git executable not found
```

### 해결법

#### 방법 1: Git Tab 재설치
Git Tab 설치 시 Git 도구도 함께 설치되는데, 설치 과정에서 문제가 있을 수 있습니다.

1. 위의 "Git Tab이 안 켜져요"에서 "재설치" 섹션을 따르세요.

#### 방법 2: 시스템 PATH 확인 (고급)

만약 시스템에 별도로 Git을 설치했다면:

1. Git이 PATH 환경 변수에 포함되어 있는지 확인합니다.
2. Windows 검색 → **"환경 변수 편집"** → "Path" 항목에 Git 폴더(예: `C:\Program Files\Git\bin`)가 있는지 확인.
3. 없다면 추가하고 재시작합니다.

## 우클릭 메뉴가 안 보여요

### 증상
Windows 탐색기에서 폴더를 우클릭했는데 **"Git Tab"** 메뉴가 없습니다.

### 해결법

#### 방법 1: Git Tab 설정에서 활성화

1. Git Tab의 **⚙ 설정** 아이콘을 클릭합니다.
2. **"통합(Integration)"** 또는 **"Windows Explorer"** 섹션을 찾습니다.
3. **"Windows 탐색기 우클릭 메뉴 활성화"** 체크박스를 **켭니다.**
4. "저장(Save)" 또는 "적용(Apply)" 버튼을 클릭합니다.

#### 방법 2: Windows 탐색기 재시작

위에서 설정을 활성화한 후:

1. 탐색기 창을 **모두 닫습니다.**
2. 탐색기를 다시 엽니다. (시작 버튼 → 탐색기 아이콘)
3. 폴더를 우클릭해봅니다.

또는:

1. `Ctrl + Shift + Esc`로 작업 관리자를 엽니다.
2. **"explorer.exe"**를 찾아 우클릭 → **"다시 시작(Restart)"**.

#### 방법 3: Windows 11 "추가 옵션" 확인

Windows 11에서는 기본 우클릭 메뉴가 간결합니다:

1. 폴더를 우클릭합니다.
2. 메뉴 최하단에 **"추가 옵션 표시(Show more options)"** 또는 **"자세히 보기"**를 클릭합니다.
3. 그러면 전체 우클릭 메뉴가 나타나고, 여기서 **"Git Tab"** 메뉴를 찾을 수 있습니다.

#### 방법 4: 다른 Git 도구의 메뉴 비활성화

TortoiseGit, GitHub Desktop 등 다른 Git GUI 도구가 우클릭 메뉴를 등록했다면 충돌할 수 있습니다.

1. 다른 도구의 설정에서 "Windows 탐색기 우클릭" 옵션을 끕니다.
2. 탐색기를 재시작합니다.
3. 필요하면 Git Tab을 설정에서 다시 활성화합니다.

## Push 인증이 실패했어요

### 증상
```
Authentication failed
Permission denied
```

### 해결법

#### 방법 1: 개인 액세스 토큰 확인

비밀번호가 아니라 **PAT(Personal Access Token)**을 사용해야 합니다.

1. [인증 설정](인증-설정) 페이지를 따라 토큰을 생성합니다.
2. Push 시 요청되는 인증창에 토큰을 입력합니다.

> ⚠️ GitHub 비밀번호는 더 이상 작동하지 않습니다!

#### 방법 2: 저장된 자격 증명 삭제 후 재시도

오래된 또는 잘못된 자격 증명이 저장되어 있을 수 있습니다.

1. Windows 검색 → **"자격 증명 관리자"** 열기.
2. **"Windows 자격 증명"** 탭 클릭.
3. GitHub 관련 항목(예: `git:https://github.com`) 찾기.
4. 클릭 → **"제거(Remove)"** 버튼 클릭.
5. Git Tab에서 다시 Push를 시도합니다.
6. 인증창이 뜨면 새 토큰을 입력합니다.

또는 Git Tab 설정에서:

1. **⚙ 설정** → **"자격 증명(Credentials)"** 섹션.
2. **"저장된 자격 증명 지우기"** 버튼 클릭.
3. 다시 시도합니다.

#### 방법 3: 토큰 권한 확인

1. GitHub 웹사이트의 Personal Access Tokens 페이지로 이동합니다.
2. 사용 중인 토큰을 확인합니다.
3. **"repo"** 권한이 체크되어 있는지 확인합니다.
4. 만료 날짜가 지나지 않았는지 확인합니다.
5. 만료됐다면 [인증 설정](인증-설정) 페이지의 "토큰 만료되었을 때" 섹션을 따르세요.

#### 방법 4: 인터넷 연결 확인

1. 인터넷이 정상인지 확인합니다.
2. 방화벽이 Git 또는 GitHub 접근을 차단하지는 않는지 확인합니다.

## 한글이 깨져요

### 증상
- 커밋 메시지나 파일명이 ????? 또는 이상한 문자로 보입니다.
- 우측 입력창에 한글을 입력할 수 없습니다.

### 해결법

#### 방법 1: 언어 설정 확인

1. **⚙ 설정** → **"Language"** (언어) 드롭다운.
2. **"한국어"** 선택.
3. 앱이 재시작되고 한글로 변환됩니다.

#### 방법 2: Windows 시스템 언어 확인

1. Windows 설정(⚙) → **"시간 및 언어"** → **"언어 및 지역"**.
2. **"Windows 표시 언어"**가 "한국어"로 설정되어 있는지 확인.
3. 영어로 되어 있다면 "한국어" 선택 후 재부팅.

#### 방법 3: 입력 환경 확인

커밋 메시지를 입력할 때 한글이 안 되면:

1. 우측 입력 박스를 클릭합니다.
2. 한글/영문 전환키(`Shift + Space` 또는 `Alt + ~`)를 누릅니다.
3. 입력기가 "한글"로 바뀌었는지 확인합니다.

## 로그/에러 메시지를 어디서 봐요?

### 증상
"뭔가 잘못된 것 같은데 자세한 오류 메시지를 보고 싶습니다."

### 해결법

Git Tab의 로그 파일 위치:

```
C:\Users\[사용자명]\AppData\Roaming\GitTab\logs
```

또는 Git Tab에서:

1. **⚙ 설정** 열기.
2. **"로그 폴더 열기"** 또는 **"Open Logs Folder"** 버튼 클릭.

로그 파일을 텍스트 에디터(메모장 등)로 열어 오류 메시지를 확인할 수 있습니다. 만약 오류를 해결하지 못했다면, 이 로그 파일을 [GitHub Issues](https://github.com/BaeTab/GitTab/issues)에 첨부해 개발자에게 보고할 수 있습니다.

## 저장소가 인식되지 않아요

### 증상
- 폴더를 열었는데 "이건 Git 저장소가 아닙니다"라는 메시지가 뜹니다.
- 우클릭 메뉴의 "열기"를 눌러도 저장소가 로드되지 않습니다.

### 해결법

#### 확인 1: 폴더에 .git 폴더가 있나?

1. Windows 탐색기에서 해당 폴더를 엽니다.
2. 상단 메뉴 → **"보기(View)"** → **"숨겨진 항목"** 체크.
3. **`.git`** 폴더가 보이는지 확인합니다.

**`.git` 폴더가 없다면:**

이 폴더는 아직 Git 저장소가 아닙니다. Git 저장소로 초기화하세요:

1. Git Tab에서 폴더를 열고 메뉴에서 **"저장소 초기화"** 또는 **"Initialize Repository"** 선택.
2. 또는 Windows 우클릭 메뉴에서 **"Git Tab"** → **"초기화"** 선택.

#### 확인 2: 폴더 권한 확인

1. 폴더를 우클릭 → **"속성(Properties)"**.
2. **"보안(Security)"** 탭에서 읽기 권한이 있는지 확인.
3. 권한이 없다면 **"편집(Edit)"**에서 사용자 권한을 추가합니다.

#### 확인 3: 경로 확인

1. 매우 긴 경로(예: 260자 이상)나 특수문자가 많은 폴더는 문제가 있을 수 있습니다.
2. 가능하면 경로를 단순하게 정리하세요.
   ```
   나쁜 예) C:\Users\user\Desktop\2024프로젝트\특별한★이름\깃저장소
   좋은 예) C:\Projects\my-repo
   ```

## 충돌(Conflict)을 해결하는 데 어려워요

### 증상
Push/Pull 또는 병합 시 "충돌이 발생했습니다"라는 메시지가 뜹니다.

### 상세 설명 및 해결법

[기능 둘러보기](기능-둘러보기) 페이지의 **"충돌 해결"** 섹션을 읽어보세요. 단계별 그림 설명이 있습니다.

만약 충돌이 복잡하다면:

1. Git Tab에서 **"중단(Abort Merge)"** 또는 **"병합 취소"** 선택.
2. 처음 상태로 돌아갑니다.
3. 팀원에게 "어느 부분을 수정했어?"라고 물어본 후 함께 수정합니다.

## 성능이 느려요 / 메모리를 많이 사용해요

### 증상
- Git Tab이 느리게 반응합니다.
- 큰 저장소를 열 때 오래 걸립니다.

### 해결법

#### 방법 1: 캐시 초기화

1. **⚙ 설정** → **"고급(Advanced)"** 섹션.
2. **"캐시 초기화"** 또는 **"Clear Cache"** 버튼 클릭.

#### 방법 2: 불필요한 저장소 제거

1. 최근 저장소 목록에서 사용하지 않는 저장소를 우클릭 → **"제거(Remove)"**.

#### 방법 3: 저장소 유지보수

정말 큰 저장소(수천 개 이상의 커밋)라면:

1. GitHub에서 저장소의 **"Settings"** → **"Danger Zone"** → **"Clean up repository"** 실행.
2. 또는 명령어를 아는 사람에게 `git gc` 명령을 요청합니다.

## 또 다른 문제가 있어요

### 해결책 순서

1. 이 페이지에서 비슷한 증상을 찾아봅니다.
2. [자주 묻는 질문](자주-묻는-질문)을 확인합니다.
3. [기능 둘러보기](기능-둘러보기)에서 관련 기능을 다시 읽어봅니다.
4. [GitHub Issues](https://github.com/BaeTab/GitTab/issues)를 검색합니다. (이미 해결된 문제일 수 있음)
5. 해결되지 않으면 [GitHub Issues](https://github.com/BaeTab/GitTab/issues)에서 **"New issue"**를 클릭해 새로운 이슈를 보고합니다.
   - 증상, 재현 방법, 로그 파일 등을 포함해서요.

## 그래도 안 되면?

1. [로그 폴더 열기](#로그에러-메시지를-어디서-봐요) 섹션을 따라 로그 파일을 확인합니다.
2. 로그 파일을 텍스트 편집기에서 열어 오류 메시지를 읽어봅니다.
3. [GitHub Issues](https://github.com/BaeTab/GitTab/issues)에 다음을 포함해 이슈를 등록합니다:
   - 증상 설명
   - 하는 행동 (단계별)
   - 로그 파일의 오류 메시지
   - Windows 버전 (설정 → 시스템 정보에서 확인)
   - .NET 런타임 버전 (`dotnet --list-runtimes` 명령, 또는 프로그램 제거에서 확인)

개발자들이 최대한 빨리 응답해드리겠습니다!
