# 인증 설정

**Push(올리기)와 Pull(받기)을 위해 GitHub/GitLab에 로그인하는 방법입니다. 처음 한 번만 설정하면 그 다음부턴 자동입니다.**

---

## 왜 비밀번호가 아니라 토큰을 쓸까?

### GitHub의 정책 변화

예전엔 GitHub 비밀번호로 로그인했습니다. 하지만 2021년부터 **GitHub은 비밀번호 인증을 없앴습니다.** 대신 **개인 액세스 토큰(Personal Access Token, PAT)**을 사용하도록 강제합니다.

### 토큰이 뭔가요?

토큰은 **일회용 암호처럼 생긴 긴 문자열**입니다. 특징:
- 비밀번호보다 안전합니다. (만료 설정 가능, 권한 제한 가능)
- 잃어버려도 비밀번호처럼 재설정할 필요가 없습니다. (토큰만 삭제하면 됨)
- Git 관련 도구들(Git Tab, VS Code 등)만 사용할 수 있도록 제한할 수 있습니다.

---

## Step 1: GitHub에서 토큰 생성하기

### GitHub 로그인
1. 브라우저에서 [github.com](https://github.com)에 접속합니다.
2. 우상단 프로필 아이콘(동그란 이미지)을 클릭합니다.
3. **"Settings"** 선택.

### 개발자 설정으로 이동
1. 좌측 메뉴를 내려 스크롤하면 **"Developer settings"** 링크가 있습니다. 클릭합니다.
2. 다시 좌측 메뉴에서 **"Personal access tokens"** → **"Tokens (classic)"** 선택.

> **참고:** 새로운 "Fine-grained tokens" 옵션도 있지만, 처음엔 "Tokens (classic)"을 사용하는 게 간편합니다.

### 토큰 생성
1. 우상단 **"Generate new token"** → **"Generate new token (classic)"** 버튼을 클릭합니다.

2. 토큰 이름을 입력합니다:
   ```
   예) GitTab
   예) GitTab-on-Windows
   ```

3. **"Expiration"** (만료)을 선택합니다:
   ```
   - No expiration: 토큰이 절대 만료되지 않음 (편하지만 덜 안전)
   - 30 days, 60 days, 90 days, 1 year: 자동 만료 (더 안전)
   ```
   
   처음이면 **"90 days"** 정도를 추천합니다. 나중에 만료되면 새 토큰을 만들면 됩니다.

4. **"Select scopes"** (권한 선택)에서 체크박스를 선택합니다:
   ```
   ✓ repo (전체 repo 접근 권한)
   ✓ read:user (사용자 정보 읽기)
   ✓ gist (Gist 접근, 선택사항)
   ```
   
   **최소한** `repo` 항목은 반드시 체크합니다.

5. 최하단 **"Generate token"** 버튼을 클릭합니다.

### 토큰 복사 (중요!)

토큰이 표시됩니다. **이 페이지를 나가면 다시 볼 수 없습니다!**

- 토큰 값 옆의 **📋 복사 버튼**을 클릭합니다.
- 또는 토큰을 직접 선택해서 `Ctrl+C`로 복사합니다.

> ⚠️ **절대 누구에게도 보여주지 마세요!** 이 토큰은 비밀번호나 다름없습니다.

## Step 2: Git Tab에서 인증하기

### 첫 Push/Pull 시도
1. Git Tab에서 **"Push"** 또는 **"Pull"** 버튼을 클릭합니다.
2. 네트워크 접근이 필요하면 **인증 대화상자**가 뜹니다.

### 인증 입력

대화상자에서:

```
사용자명: your_github_username
토큰:    ghp_xxxxxxxxxxxxxxxxxxxxxxxxxxxx (위에서 복사한 것)
```

- **사용자명:** GitHub 계정 이름 (예: `john_doe`)
- **토큰:** 위에서 복사한 긴 문자열

"확인" 또는 "로그인" 버튼을 클릭합니다.

### 자동 저장 (Windows 자격 증명)

인증 정보가 **Windows 자격 증명 관리자**에 안전하게 저장됩니다.

다음부터는:
- Push/Pull 시 자동으로 인증됩니다.
- 다시 입력할 필요 없습니다.

---

## Step 3: 동작 확인

### Push 테스트
1. 파일을 수정하고 커밋합니다.
2. **"Push"** 버튼을 클릭합니다.
3. 잠깐 후 "성공" 메시지가 나타나면 GitHub에 올라갔습니다.
4. 브라우저에서 `github.com/당신의_계정/저장소이름`으로 가서 커밋이 보이는지 확인합니다.

### Pull 테스트
1. GitHub 웹사이트에서 파일을 수정합니다.
2. 커밋 메시지를 입력하고 **"Commit"** 버튼을 클릭합니다.
3. Git Tab으로 돌아와 **"Pull"** 버튼을 클릭합니다.
4. 파일이 자동으로 업데이트됩니다.

## 저장된 자격 증명 관리

### 저장된 자격 증명 보기
1. Windows 검색창(🔍)에서 **"자격 증명"** 입력.
2. **"자격 증명 관리자"** 앱을 엽니다.
3. **"Windows 자격 증명"** 탭을 클릭합니다.
4. GitHub 관련 항목들이 나열됩니다.

### 자격 증명 삭제
1. 삭제하고 싶은 항목을 클릭합니다.
2. **"제거"** 버튼을 클릭합니다.
3. 다음 Push/Pull 시 다시 입력하라는 창이 뜹니다.

### Git Tab 설정에서 관리
1. Git Tab의 **⚙ 설정**을 엽니다.
2. **"인증(Authentication)"** 섹션을 찾습니다.
3. **"저장된 자격 증명 지우기"** 버튼을 클릭하면, 지금 열려 있는 저장소의 원격 호스트에 저장된 자격 증명이 삭제됩니다. (다음 Push/Pull 때 다시 입력하게 됩니다.)

## 토큰 만료되었을 때

### 증상
Push/Pull 시 인증 오류가 났습니다:
```
error: authentication failed
```

### 해결법

1. GitHub 웹사이트에서 새 토큰을 생성합니다. ([Step 1](#step-1-github에서-토큰-생성하기) 참고)

2. 자격 증명 관리자에서 오래된 토큰을 삭제합니다. ([위 항목](#자격-증명-삭제) 참고)

3. Git Tab에서 다시 Push/Pull을 시도합니다.

4. 새 토큰을 입력합니다.

---

## GitLab 또는 다른 서비스 사용

같은 방식으로 작동합니다.

### GitLab의 경우

1. GitLab 웹사이트(`gitlab.com`)에 로그인합니다.

2. 우상단 프로필 → **"Edit profile"** → **"Access tokens"** 클릭.

3. **"Add new token"** 버튼 클릭.

4. 토큰 이름을 입력하고 권한 중 `api`와 `read_repository`, `write_repository` 체크.

5. **"Create personal access token"** 버튼 클릭.

6. 토큰을 복사합니다.

7. Git Tab에서 Push/Pull 시 
   - **사용자명:** GitLab 사용자명 또는 이메일
   - **토큰:** 복사한 GitLab 토큰
   
   을 입력하면 됩니다.

## FAQ: 인증 관련

### Q: 토큰을 잃어버렸어요.
**A:** 다시 만들면 됩니다. GitHub에 로그인 후 "Personal access tokens"에서 확인할 수 있습니다. 예전 토큰은 보이지 않지만, "Generate new token"으로 새로 만들 수 있습니다.

### Q: 여러 GitHub 계정을 사용하는데요.
**A:** Git Tab은 자격 증명을 **호스트별**(예: `github.com`)로 저장합니다. 계정을 바꾸려면 설정의 **"저장된 자격 증명 지우기"**로 기존 정보를 지운 뒤, 다음 Push/Pull 때 다른 계정의 사용자명·토큰을 입력하면 됩니다. (또는 Windows "자격 증명 관리자"에서 직접 항목을 바꿀 수 있어요.)

### Q: 회사 GitHub(GitHub Enterprise)를 쓰는데요.
**A:** 같은 방식입니다. 단지 GitHub 주소가 `github.com`이 아니라 회사 도메인(예: `github.company.com`)일 뿐입니다.

### Q: 토큰이 나갔어요. 응급입니다!
**A:** GitHub 웹사이트에서 해당 토큰을 즉시 삭제합니다. Settings → Developer settings → Personal access tokens에서 삭제 버튼을 클릭하면 그 토큰은 더 이상 사용 불가능합니다.
