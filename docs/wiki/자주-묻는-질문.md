# 자주 묻는 질문 (FAQ)

**Git Tab 사용자들이 가장 많이 하는 질문 10가지입니다.**

---

## Q1: Git을 꼭 설치돼 있어야 하나요?

**A:** 아닙니다! Git Tab 설치 시 필요한 Git 도구가 **자동으로 번들로 포함**됩니다. 따로 설치할 필요가 없습니다.

- Git Tab만 설치하면 바로 사용할 수 있습니다.
- 명령어 창(CMD, PowerShell, Git Bash)도 필요 없습니다.
- 마우스만으로 모든 Git 작업을 할 수 있습니다.

---

## Q2: Git Tab은 정말 무료인가요?

**A:** 네, 100% 무료 **오픈소스** 소프트웨어입니다.

- 구독료나 라이선스 비용이 없습니다.
- 광고도 없습니다.
- GitHub에 소스 코드가 공개되어 있습니다.
- 누구나 자유롭게 사용, 배포, 수정할 수 있습니다.

---

## Q3: 내 코드가 외부로 전송되거나 저장되지 않나요?

**A:** 완전히 안전합니다.

- Git Tab은 **당신의 컴퓨터에서만** 작동합니다. (로컬 앱)
- 코드가 외부 서버로 자동 전송되지 않습니다.
- GitHub/GitLab에 올리려면 당신이 **직접 "Push" 버튼을 클릭**해야 합니다.
- 개인 액세스 토큰도 Windows 자격 증명에만 저장됩니다. (오프라인 상태)

---

## Q4: 커밋을 잘못했는데 어떻게 되돌릴까요?

**A:** 여러 방법이 있습니다.

### 방법 1: 마지막 커밋 수정하기 (Amend)
마지막 커밋이 실수였다면:
1. 우측 "변경사항" 섹션에서 파일을 수정합니다.
2. 커밋 메시지를 바꾸고 싶다면 새 메시지를 입력합니다.
3. **"Amend"** 체크박스를 켭니다.
4. "커밋" 버튼을 클릭합니다.

마지막 커밋이 수정됩니다.

### 방법 2: 커밋 되돌리기 (Reset)
1. 커밋을 우클릭합니다.
2. **"Reset"** 선택.
3. 버전을 선택합니다:
   - **Soft:** 파일은 그대로, 커밋만 취소
   - **Mixed:** 파일도 수정 전 상태로
   - **Hard:** 모든 변경사항 완전 삭제 (주의!)

### 방법 3: Revert (새 커밋으로 되돌리기)
과거 커밋을 되돌려야 한다면:
1. 커밋을 우클릭 → **"Revert"** 선택.
2. 새로운 "되돌림" 커밋이 생성됩니다.
3. 이 방법은 이미 GitHub에 올린 커밋도 안전하게 처리할 수 있습니다.

---

## Q5: 실수로 파일을 삭제했어요!

**A:** 다시 복구할 수 있습니다.

### 방법 1: 최근 커밋으로부터 복구
1. 좌측 커밋 그래프에서 파일이 있던 시점의 커밋을 찾습니다.
2. 커밋을 우클릭 → **"Checkout"** 또는 **"파일 복구"** 선택.
3. 파일이 복구됩니다.

### 방법 2: 파일 목록에서 직접 복구
1. 우측 "변경사항"에서 삭제된 파일을 찾습니다.
2. 우클릭 → **"되돌리기(Discard)"** 선택.
3. 마지막 커밋 상태로 파일이 복구됩니다.

### 중요한 파일을 삭제했다면?
Git은 모든 커밋 이력을 보관합니다. 몇 년 전 커밋에서도 파일을 복구할 수 있습니다!

## Q6: 브랜치가 뭔가요?

**A:** [Git 기초](Git-기초) 페이지에서 자세히 설명했지만, 간단히 말하면:

**브랜치 = 평행 작업 공간**

마치 게임에서 여러 세이브 파일로 다른 엔딩을 시도하듯이, 메인 코드를 건드리지 않고 새 기능을 따로 개발할 수 있습니다.

```
main ────o────o────o  (메인 코드)
         \
          o──o──o  (새 기능 브랜치)
               \
                o──o (다른 기능)
```

## Q7: Push가 안 돼요!

**A:** 가장 흔한 원인 3가지:

### 원인 1: 인증 정보가 없음
- GitHub/GitLab에 아직 로그인하지 않았습니다.
- **해결:** [인증 설정](인증-설정) 페이지를 따라 개인 액세스 토큰을 설정하세요.

### 원인 2: 커밋이 아무것도 없음
- 파일을 수정했지만 아직 커밋하지 않았습니다.
- **해결:** 우측에서 파일을 선택하고 커밋 메시지를 입력한 후 "커밋"을 클릭하세요.

### 원인 3: 네트워크 연결이 없음
- 인터넷이 끊겼을 수 있습니다.
- **해결:** 인터넷 연결을 확인하고 다시 시도하세요.

더 자세한 내용은 [문제 해결](문제-해결) 페이지를 확인하세요.

## Q8: 한국어/영어를 바꾸려면?

**A:** 아주 간단합니다.

1. 우상단 **⚙ 설정** 아이콘을 클릭합니다.
2. **"Language"** (언어) 드롭다운을 찾습니다.
3. **"한국어"** 또는 **"English"** 선택.
4. 앱이 자동으로 재시작되고 언어가 바뀝니다.

---

## Q9: 테마를 라이트에서 다크로 (또는 반대로) 바꾸려면?

**A:** 마찬가지로 간단합니다.

1. **⚙ 설정** 아이콘을 클릭합니다.
2. **"Theme"** (테마) 섹션을 찾습니다.
3. **"Light"** (라이트) 또는 **"Dark"** (다크) 선택.
4. 앱 색상이 즉시 변경됩니다.

> **팁:** 다크 테마는 밤 작업할 때 눈이 피로하지 않습니다.

---

## Q10: Git Tab을 업데이트하려면?

**A:** 자동 업데이트가 지원됩니다.

### 자동 업데이트 (권장)
- Git Tab의 **⚙ 설정**에서 **"자동 업데이트"** 또는 **"Auto Update"** 옵션이 있습니다.
- 이를 켜면 새 버전이 나왔을 때 자동으로 다운로드·설치됩니다.
- 재시작하라는 메시지가 뜬 후 "재시작"을 클릭하면 최신 버전이 됩니다.

### 수동 업데이트
- 자동 업데이트를 끈 경우:
1. [GitHub Releases 페이지](https://github.com/BaeTab/GitTab/releases)에 방문합니다.
2. 최신 버전의 설치 파일을 다운로드합니다.
3. 파일을 실행해 다시 설치합니다.
4. 기존 설정은 그대로 유지됩니다.
