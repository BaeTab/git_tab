# Windows 탐색기 우클릭

**Windows 탐색기(File Explorer)에서 아무 폴더나 우클릭만 해도 Git 명령을 바로 실행할 수 있습니다. 정말 편합니다!**

---

## 기능 활성화하기

### 1단계: Git Tab 설정 열기
1. Git Tab을 실행합니다.
2. 우상단 **⚙ 설정(Settings)** 아이콘을 클릭합니다.

### 2단계: 탐색기 통합 켜기
1. 설정 창의 왼쪽 메뉴에서 **"통합(Integration)"** 또는 **"Windows 탐색기"** 항목을 찾습니다.
2. **"Windows 탐색기 우클릭 메뉴 활성화"** 또는 **"Enable Explorer Context Menu"** 체크박스를 켭니다.
3. "저장(Save)" 또는 "적용(Apply)" 버튼을 클릭합니다.

### 3단계: Windows 탐색기 새로고침 (선택사항)
탐색기가 이미 열려 있다면:
- 탐색기 창을 한 번 닫았다가 다시 열거나,
- `Ctrl + Shift + Esc` 또는 작업 관리자에서 탐색기(explorer.exe)를 다시 시작합니다.

> **팁:** 재부팅하면 100% 확실하게 반영됩니다.

---

## 우클릭 메뉴 사용하기

### Git Tab이 없는 폴더에서 우클릭

아무 폴더나 우클릭하면 메뉴 어딘가에 **"Git Tab"** 또는 **"Git"** 항목이 나타납니다.

> **Windows 11 사용자 주의:** "추가 옵션 표시"를 클릭해야 하는 경우도 있습니다. (오른쪽 클릭 → 최하단 "추가 옵션 표시")

### 메뉴 항목들 (총 8개)

메뉴를 펼치면 다음과 같은 옵션들이 나타납니다:

#### 📂 **열기 (Open)**
- Git Tab이 시작되고 해당 폴더가 저장소로 로드됩니다.
- 이미 Git 저장소라면 커밋 이력이 표시됩니다.
- 아직 Git 저장소가 아니라면 폴더 구조만 표시되고, Git Tab 메뉴에서 "저장소 초기화"를 선택해 새 저장소를 만들 수 있습니다.

#### 🆕 **복제 (Clone)**
- GitHub, GitLab 같은 원격 저장소의 URL을 입력하면 내 PC로 복제합니다.
- URL을 입력하면 자동으로 선택한 폴더 아래에 저장소를 복제합니다.
- 인증이 필요하면 사용자 이름 + 토큰을 입력합니다.

#### 📝 **커밋 (Commit)**
- 별도 대화상자가 열립니다.
- 변경된 파일들이 자동으로 표시됩니다.
- 커밋 메시지를 입력하고 "커밋" 버튼을 클릭합니다.
- 메인 앱을 열지 않고 바로 커밋할 수 있어서 빠릅니다!

#### ⬇️ **풀 (Pull)**
- 별도 대화상자가 열립니다.
- "풀" 버튼을 클릭하면 서버의 최신 코드를 다운로드합니다.
- 인증이 필요하면 사용자 이름 + 토큰을 입력합니다.

#### ⬆️ **푸시 (Push)**
- 별도 대화상자가 열립니다.
- 로컬 커밋들을 GitHub/GitLab으로 올립니다.
- 인증이 필요하면 사용자 이름 + 토큰을 입력합니다.

#### 🔄 **페치 (Fetch)**
- 별도 대화상자가 열립니다.
- 서버의 최신 정보만 확인하고(파일은 변경 안 함) 대화상자가 닫힙니다.

#### 📦 **변경 스태시 (Stash)**
- 작업 중인 파일들을 임시로 보관합니다.
- 별도 대화상자가 열려 스태시에 이름을 붙일 수 있습니다 (선택사항).
- 현재 작업을 저장하고 다른 브랜치로 전환해야 할 때 유용합니다.

#### 📜 **로그 (Log)**
- Git Tab이 메인 창으로 열리고 커밋 이력이 표시됩니다.
- "열기"와 비슷하지만 이력을 더 자세히 보는 데 최적화됐습니다.

## 예제: 전형적인 작업 흐름

### 시나리오: 코드를 수정하고 GitHub에 올리기

1. **코드 수정**  
   텍스트 에디터(VS Code, Visual Studio 등)에서 코드를 수정합니다.

2. **탐색기에서 우클릭**  
   변경한 폴더를 탐색기에서 우클릭 → "Git Tab" → "커밋" 선택.

3. **메시지 입력 & 커밋**  
   커밋 대화상자에서 메시지를 입력합니다.
   ```
   예) "로그인 폼 유효성 검사 추가"
   ```
   "커밋" 버튼을 클릭합니다.

4. **GitHub에 올리기**  
   같은 폴더를 다시 우클릭 → "Git Tab" → "푸시" 선택.
   "푸시" 버튼을 클릭합니다.

5. **완료!**  
   대화상자가 닫히면 코드가 GitHub에 올라갔습니다.

> **팁:** 처음 푸시할 때만 인증을 요구합니다. 그 다음부턴 자동으로 인증됩니다.

## 우클릭 메뉴가 안 보일 때

### 확인해야 할 것들

**1. 설정에서 정말 활성화했나?**
- Git Tab 설정(⚙)에서 "Windows 탐색기 우클릭" 옵션을 다시 확인합니다.

**2. 탐색기를 다시 시작했나?**
- 탐색기 창을 모두 닫고 다시 열어봅니다.
- 또는 재부팅합니다.

**3. Windows 11을 사용하나?**
- 우클릭 메뉴 → 최하단 **"추가 옵션 표시"** 클릭.
- 그러면 전체 우클릭 메뉴가 나타납니다.

**4. 관리자 권한이 필요한 건 아닐까?**
- Git Tab을 관리자로 재실행해봅니다 (보통 필요 없습니다).

**5. 다른 Git GUI 도구의 우클릭 메뉴가 있나?**
- TortoiseGit, GitHub Desktop 등이 우클릭 메뉴를 등록했다면 충돌할 수 있습니다.
- 다른 도구의 메뉴를 비활성화했다가 Git Tab을 다시 설정해봅니다.

### 여전히 안 보이면?

[문제 해결](문제-해결) 페이지의 **"우클릭 메뉴가 안 보여요"** 항목을 확인하세요.

## 커밋/푸시/풀/페치 대화상자

### 커밋 대화상자

```
┌─────────────────────────────────────┐
│  커밋 대화상자                       │
├─────────────────────────────────────┤
│ 변경된 파일:                         │
│ ☑ src/main.py (23 changes)         │
│ ☑ README.md (2 changes)            │
│ ☐ .gitignore (unchanged)           │
│                                     │
│ 커밋 메시지:                        │
│ ┌─────────────────────────────────┐ │
│ │ 로그인 기능 구현                │ │
│ └─────────────────────────────────┘ │
│                                     │
│ [커밋]  [취소]                      │
└─────────────────────────────────────┘
```

- 커밋하고 싶은 파일만 체크박스를 켭니다.
- 메시지를 입력합니다.
- "커밋" 버튼을 클릭합니다.

### 푸시/풀/페치 대화상자

```
┌──────────────────────────┐
│ 푸시 대화상자            │
├──────────────────────────┤
│ 브랜치: main             │
│ 원격: origin             │
│                          │
│ 준비 중...               │
│                          │
│ [푸시]  [취소]           │
└──────────────────────────┘
```

- 자동으로 현재 브랜치가 선택됩니다.
- "푸시/풀/페치" 버튼을 클릭합니다.
- 인증이 필요하면 사용자 이름 + 토큰을 입력하라는 창이 뜹니다.

---

## 관리자 권한이 필요한가?

**아닙니다!** 우클릭 메뉴 통합은 관리자 권한 없이 설정할 수 있습니다. 

Windows 레지스트리에 등록하는 과정이지만, Git Tab이 자동으로 처리합니다. (관리자 권한 필요 없음)
